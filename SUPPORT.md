# Support

## How to Get Help

AISquare Studio AutoQA is an open-source project. We are happy to help you get started and answer questions.

### Documentation

Before opening an issue, please check the existing documentation:

- [README.md](README.md) — Quick start and usage guide
- [docs/ACTION_USAGE.md](docs/ACTION_USAGE.md) — Detailed action usage
- [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) — System architecture
- [docs/AUTOQA_QUICK_START.md](docs/AUTOQA_QUICK_START.md) — Quick start guide
- [docs/ACTIVE_EXECUTION.md](docs/ACTIVE_EXECUTION.md) — Active execution mode
- [docs/LINTING.md](docs/LINTING.md) — Linting configuration

### Questions and Discussions

- **GitHub Issues:** For bug reports and feature requests, please use our [issue templates](https://github.com/AISquare-Studio/AISquare-Studio-QA/issues/new/choose)
- **GitHub Discussions:** For questions, ideas, and general conversation (when enabled)

### Bug Reports

If you've found a bug, please [open an issue](https://github.com/AISquare-Studio/AISquare-Studio-QA/issues/new?template=bug_report.md) with:

- A clear description of the problem
- Steps to reproduce the issue
- Expected vs. actual behavior
- Relevant logs or screenshots

### Feature Requests

Have an idea for improvement? [Open a feature request](https://github.com/AISquare-Studio/AISquare-Studio-QA/issues/new?template=feature_request.md) with:

- A clear description of the feature
- The problem it solves
- Any alternatives you've considered

### Security Issues

**Do not report security vulnerabilities through public issues.** Please see our [Security Policy](SECURITY.md) for responsible disclosure instructions.

## Contributing

Want to contribute? Check out our [Contributing Guide](CONTRIBUTING.md) to get started.
